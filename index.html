Basics

JavaScript Fundamentals

What is the difference between var, let, and const?

var: Function-scoped and can be redeclared, leading to potential bugs. let: Block-scoped, ideal for variables that may change. const: Block-scoped, used for variables that shouldn’t be reassigned.

Explain the event loop in JavaScript.

The event loop ensures non-blocking execution in JavaScript by handling asynchronous operations. It moves completed tasks from the callback queue to the call stack when the stack is empty.

What is the difference between == and === in JavaScript?

== compares values after type conversion (loose equality). === compares values and types (strict equality).

What is a closure in JavaScript?

A closure is a function that retains access to its outer lexical scope, even after the outer function has executed.

function outer() { let count = 0; return function inner() { count++; return count; }; } const counter = outer(); console.log(counter()); // 1 console.log(counter()); // 2

How does async/await improve asynchronous programming?

async/await makes asynchronous code easier to read and maintain by allowing you to write it in a synchronous style. It is built on top of Promises.

React Basics

What are React hooks, and why are they used?

A: React hooks, like useState and useEffect, allow function components to manage state and side effects, making code more concise and reusable.

How does the virtual DOM work?

The virtual DOM is a lightweight copy of the actual DOM. React updates the virtual DOM first, calculates the difference (diffing), and applies only the changes to the real DOM for better performance.

How do you manage state in a React application?

Component State: Using useState or useReducer for local state. Global State: Libraries like Redux, Context API, or Zustand

What is the difference between controlled and uncontrolled components in React?

Controlled Component: The form input's value is controlled by React state. Uncontrolled Component: The DOM maintains the input's state, and refs are used to access its value.

Angular Basics

What is two-way data binding in Angular?

Two-way data binding ensures synchronization between the model and the view. Changes in the UI automatically update the data and vice versa using the [(ngModel)] directive.

What are services in Angular?

Services are singleton objects that enable data sharing and logic reuse across components in Angular, typically implemented with dependency injection.

What is dependency injection in Angular?

Dependency injection (DI) is a design pattern where Angular injects required dependencies (e.g., services) into components automatically, improving modularity and testing.

How does Angular handle routing?

Angular uses the RouterModule to configure routes. You define routes in the app-routing.module.ts file and navigate between them using routerLink or programmatically via the Router service.

Theoretical Questions

What is the difference between synchronous and asynchronous programming?

A: Synchronous: Code is executed line-by-line, blocking further execution until the current task is finished. Asynchronous: Code executes independently, allowing non-blocking operations via callbacks, promises, or async/await.

How do RESTful APIs work?

RESTful APIs follow the principles of REST, using HTTP methods like GET, POST, PUT, and DELETE to perform CRUD operations on resources identified by URLs.

Explain the concept of REST in web development.

REST (Representational State Transfer) is an architectural style that uses standard HTTP methods like GET, POST, PUT, and DELETE to interact with resources represented by URLs.

What is the difference between client-side and server-side rendering?

Client-side rendering (CSR): JavaScript renders content in the browser after downloading. Server-side rendering (SSR): The server generates the full HTML for a page and sends it to the browser, improving SEO and initial load times.

Practical Angular/React Scenarios

How would you optimize a React app for performance?

Use React.memo for component re-renders. Optimize state management by colocating state. Use lazy loading for routes and components. Implement code splitting with tools like Webpack.

How would you implement error handling in an Angular application?

Use interceptors for HTTP error handling. Provide user feedback with alerts or modals. Implement a global error handler using the ErrorHandler class.

Debugging and Problem-Solving

How do you debug a complex issue in JavaScript?

Reproduce the issue in isolation. Use debugging tools like Chrome DevTools. Log key variables to track data flow. If necessary, write unit tests to pinpoint the bug.

If a React component isn’t rendering correctly, how would you troubleshoot?

Check props and state for errors. Ensure parent components pass correct data. Validate lifecycle methods or hooks usage. Inspect for potential syntax or logic errors.

How can you improve the performance of a web application?

Minimize DOM manipulations. Use lazy loading for assets. Optimize images and static files. Avoid unnecessary re-renders in React using React.memo or useCallback.
